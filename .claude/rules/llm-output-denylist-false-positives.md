# LLM 응답에 금칙어/denylist 필터를 걸 때

자유 텍스트를 생성하는 LLM 응답에 fail-closed 금칙어 검사를 붙일 때는 다음을 지킨다.

## 규칙

- **금칙어는 의도가 명확한 구문 단위로 좁힌다.** 단일 단어(예: "매수", "매도")는 도메인에서 완전히 정상적인 문맥(시장 흐름 리포팅 등)에도 흔히 등장해 오탐률이 높다. "매수 추천", "매도 시그널", "목표가"처럼 실제로 금지하려는 의도(투자 권유 등)가 구문 자체에 담긴 표현만 금칙어로 쓴다.
- **필터를 추가하면, 그 필터가 정상 응답을 걸러내지 않는지 확인하는 회귀 테스트를 반드시 같이 추가한다.** "금지어가 있으면 막힌다"는 테스트만으로는 부족하다 — "이 도메인에서 흔히 쓰이는 정상 문장은 안 막힌다"도 검증해야 오탐을 잡는다.
- **로컬에서 몇 번 성공했다고 안전하다고 판단하지 않는다.** LLM 응답은 매번 문구가 달라지므로, 로컬 테스트가 통과한 것은 우연히 그 단어가 안 걸린 샘플만 관찰했을 뿐일 수 있다. 응답 텍스트의 분량이 늘어나거나(지표 추가 등) 프롬프트가 바뀌면 오탐 확률이 급격히 올라갈 수 있다.
- **배포 직후 실제 환경에서 한 번 더 실행 경로를 확인한다.** 특히 fail-closed 검증 로직은 "막혀야 할 때 막히는가"뿐 아니라 "막히지 않아야 할 때 안 막히는가"까지 실제 트래픽으로 확인해야 한다.

## 배경

`artifacts/investment-4-seasons/learnings.md`에 기록된 사례: 지표를 3개→7개로 늘리며 응답 텍스트가 길어지자, "매수"/"매도" 단독 단어를 막던 금칙어 필터가 정상적인 채권·환율 시장 리포팅 문장("외국인 매도세로 금리 상승")에도 매번 걸려 프로덕션에서 API 요청 100%가 실패했다. 로컬에서는 우연히 그 단어가 안 걸리는 응답만 관찰되어 발견하지 못했다.
