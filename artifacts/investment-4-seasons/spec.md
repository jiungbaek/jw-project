# AI 투자 4계절 웨더캐스터 — spec.md

## 개요
사용자가 메인 페이지에서 버튼 하나를 누르면, Gemini의 실시간 검색(grounding)이 오늘 기준 CPI·미국채 금리·주요 지수 추세를 파악해 현재 매크로 국면을 봄/여름/가을/겨울 4계절 중 하나로 판정하고, 판정 근거와 일반적 자산 경향 정보를 결과 페이지에 보여준다. 크롤러·DB·로그인 없이 완전 stateless로 동작한다.

## 범위

### 포함
- 메인 페이지: 서비스 소개 + "오늘의 계절 확인하기" CTA
- 결과 페이지: 계절 판정(봄/여름/가을/겨울 중 1개) + 지표별(CPI/금리/지수) 한 줄 판정 근거 + 계절별 일반 자산군(성장주/가치주/원자재/채권 등) 경향 정보
- 요청마다 Gemini 실시간 검색을 새로 호출 (캐싱 없음, 재요청 시 결과가 달라질 수 있음을 허용)
- Gemini 호출 실패/응답 파싱 실패 시 에러 안내 + 재시도 버튼

### 제외
- 크롤링/자체 데이터 파이프라인 — Gemini 실시간 검색이 유일한 데이터 소스이므로 불필요
- 로그인, 사용자별 이력 저장, DB — stateless 데모 목적에 불필요
- 결과 캐싱 — 매 요청 새로 판정하기로 결정
- 특정 종목명·매수/매도 시그널·목표가 추천 — 자산군(주식/채권/금/원자재) 단위보다 더 구체적인 실행 지시는 스코프 밖. 재미 콘텐츠 톤을 유지하기 위해 비조언 고지 문구는 넣지 않기로 결정
- SNS 공유 카드 이미지 — 3시간 스코프 밖, 시간 남으면 다음 사이클에서 검토

## 시나리오

### 1. 정상 판정 흐름
- **Given** 사용자가 메인 페이지에 접속함
- **When** "오늘의 계절 확인하기" 버튼을 클릭함
- **Then** 결과 페이지로 이동하고, 로딩 표시 후 4계절(봄/여름/가을/겨울) 중 하나의 판정 결과와 지표별 한 줄 근거, 계절별 일반 자산군 경향 정보가 표시된다

성공 기준:
- [ ] 버튼 클릭 → 결과 페이지로 이동하고 로딩 상태가 보인다
- [ ] 판정 완료 → "봄"/"여름"/"가을"/"겨울" 중 정확히 하나의 계절명이 화면에 표시된다
- [ ] 판정 완료 → CPI, 금리, 지수 각각에 대한 한 줄 요약이 표시된다 (3개 항목)
- [ ] 판정 완료 → 판정된 계절에 해당하는 자산군(성장주/가치주/원자재/채권 등) 경향 문구가 표시된다

### 2. Gemini 호출/파싱 실패
- **Given** 사용자가 "오늘의 계절 확인하기" 버튼을 클릭해 결과 페이지로 이동함
- **When** Gemini 호출이 실패하거나 응답을 계절 판정으로 파싱할 수 없음
- **Then** 계절 판정 대신 에러 안내 문구와 "다시 시도" 버튼이 표시된다

성공 기준:
- [ ] Gemini 호출 실패 → "지금 계절을 파악하기 어렵습니다" 류의 에러 안내 문구가 표시된다
- [ ] 에러 상태 → "다시 시도" 버튼이 표시된다
- [ ] "다시 시도" 버튼 클릭 → Gemini 호출이 재시도되고 로딩 상태로 전환된다

### 3. 재방문 시 결과 변동 허용
- **Given** 사용자가 이미 한 번 판정 결과를 본 상태
- **When** 메인 페이지로 돌아가 "오늘의 계절 확인하기"를 다시 클릭함
- **Then** 새로운 Gemini 호출이 실행되고, 이전 판정과 다른 계절이 나와도 정상 동작으로 취급된다 (캐시된 이전 결과를 보여주지 않음)

성공 기준:
- [ ] 재클릭 → 새 로딩 상태가 다시 나타난다 (이전 결과 화면이 즉시 재사용되지 않음)

## 불변 규칙

- 결과 화면에 특정 종목명, 매수/매도 시그널, 목표가는 어떤 경로로도 표시되지 않는다 — 계절별 자산군(주식/채권/금/원자재 등) 단위의 일반적 경향 서술만 허용

## 의존성
